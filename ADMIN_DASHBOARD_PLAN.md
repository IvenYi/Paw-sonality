# Admin Dashboard 功能规划与设计 (商业化 MVP)

> **目标**: 将隐蔽的 `/admin` 路由打造为具备数据洞察与客服能力的“多邻国风 (Bubbly UI)”运营中枢，重点收集用户 4 轴及 5 轴宠格数据反哺小红书引流。

---

## 1. 核心模块规划 (4 大看板)

### 📊 A. 转化概览看板 (Conversion Overview)
用可爱的 Q 弹卡片展示当日最重要的北极星指标：
- **总发卡数**: 当前生成的激活码总量。
- **累计核销率**: `已核销数 / 总数` (带进度条)。
- **今日进账**: 今日新增核销数（直观反映小红书笔记的当日带货能力）。

### 🎯 B. 宠格引流素材库 (Marketing Insights)
这是你发下一篇小红书爆款的数据源：
- **猫狗阵营对决**: 饼图或横向占比条，直观展示买单的养猫多还是养狗多。
- **全网宠格排行榜 (TOP 5)**: 列出最高频出现的 MBTI (如 `ENFP-T 24只`, `INTJ-A 10只`)。
- *数据收集策略*：用户做完 12 题，触发“解锁码校验”成功瞬间，先记录基础 4 轴结果 (`base_mbti`)；待用户做完 4 道深层题到达 `FINAL` 阶段，再次调用 API 补全第 5 维度 (`final_mbti`)。

### 🔍 C. 客服与订单管理 (Customer Service)
解决用户“我的码用不了”的扯皮问题：
- **极速搜索框**: 输入 6 位短码实时过滤列表。
- **订单详情回溯**:
  - 状态：未使用 / 已核销。
  - 核销时间：`2026-04-21 14:30:00`。
  - 绑定设备摘要：`a1b2c3d4...` (用于证明码是在别人的手机上被用的)。
  - 测试结果追溯：TA 家的是猫还是狗，最终测出了什么。
- **一键作废/重置**: 遇到特殊退款或确实需要解绑设备的客诉，提供一个 `Reset` (解绑设备) 或 `Invalidate` (彻底作废) 的操作按钮。

### 📦 D. 批次 ROI 追踪 (Batch Tracking)
- 支持按 `batch_id` (发卡批次) 查看或生成。
- 直观对比不同小红书笔记挂的码的核销情况。

## 2. 数据库结构升级 (Supabase)

为支持数据分析，我们需要对 `redemption_codes` 表追加字段：

| 新增字段 | 类型 | 描述 |
| :--- | :--- | :--- |
| `pet_type` | varchar(10) | `dog` 或 `cat` (在首次核销时写入) |
| `base_mbti` | varchar(4) | 如 `ENFP` (在首次核销时写入) |
| `final_mbti` | varchar(6) | 如 `ENFP-T` (在深层题做完后追写) |
| `owner_mbti` | varchar(4) | 主人性格 (在深层题做完后追写) |

---

## 3. 技术栈与 UI 方案
- **UI 风格**: 延续现有的 Bubbly UI，使用 `framer-motion` 做数据加载时的果冻回弹效果，`lucide-react` 补充管理端图标 (Chart, Search, ShieldAlert)。
- **数据获取**: 在 `src/api/verification.ts` 中新增获取聚合数据 (Count, GroupBy) 的只读接口。
- **状态流转**: `ResultPage.tsx` 在原有的 `verifyRedemptionCode` 基础上，增加一个 `updateFinalResult` 的方法，用于追记 5 轴数据。
